# LiteRehab 详细 Demo 指南设计

## 目标

把现有 `DEMO_GUIDE.md` 更新成一份可在课程答辩现场直接照着执行的 6–8 分钟演示指南。主线只演示右臂，覆盖正常动作、错误动作、视觉融合、相机降级和数据保存。

## 演示范围

- 默认使用右臂与右前臂佩戴方式，不要求重复演示左臂。
- MaixCAM2 使用 RTSP over USB NCM，默认地址为 `rtsp://10.203.102.1:8554/live`。
- 演示两种受固件支持的动作：肘部屈伸和前臂旋转。
- 演示三种训练反馈：动作过快、幅度不足和躯干代偿。
- 遮挡相机以展示 `Fusion` 到 `IMU Only` 的安全降级，并在恢复画面后返回 `Fusion`。
- 结束时正常退出 Dashboard 并确认 CSV 已保存。

## 佩戴与站位

- MPU6050 固定在右前臂背侧，位于腕横纹近端约 5–8 cm，不能压在腕关节或只绑在手背上。
- 传感器 X 轴沿前臂长轴指向手部，Z 轴垂直皮肤朝外；绑带应牢固但不影响循环或动作。
- MaixCAM2 横向放置在参与者正前方约 1.5–2.0 m，镜头高度接近胸口。
- 画面必须包含右肩、右肘、右腕和右髋，并尽量保留整个上半身。

## 文档结构

1. 每次启动的命令：设备检查、MaixCAM2 脚本、Dashboard、完整测试和必要的重启命令。
2. 现场硬件布局与传感器佩戴：位置、方向、松紧度和校准要求。
3. 是否需要左右手：说明右臂为正式主线，左臂仅作可选扩展。
4. 6–8 分钟逐步演示脚本：每一步包含操作、建议口播、预期 OLED/LED/蜂鸣器/Dashboard 结果。
5. 成功判定表：串口、摄像头、Fusion、动作状态、计数、反馈和 CSV。
6. 快速故障处理：RTSP、串口占用、BLE、关键点不可见和 IMU 校准问题。
7. 结束与安全说明。

## 动作顺序与预期结果

1. 中立位设置基线：按 `b`，Dashboard 保持 `Ready`。
2. 正常肘部屈伸：2–3 秒完成屈曲和返回；显示 `Elbow Flexion`，成功时计数增加并发出一声高音。
3. 正常前臂旋转：肘部约 90°、上臂固定，掌心内外旋后返回；显示 `Forearm Rotation`，成功时计数增加并发出一声高音。
4. 过快动作：完整但快速地完成一次往返；提示 `SLOW DOWN`/`too_fast`，发出两声低音。
5. 幅度不足：完成小幅往返；提示 `INCREASE RANGE`/`insufficient_range`，发出两声低音，固件计数可能不增加。
6. 躯干代偿：屈肘时明显侧倾躯干；视觉提示 `AVOID TRUNK COMPENSATION`。
7. 相机遮挡与恢复：遮挡后显示 `IMU Only`，IMU 数据和反馈继续；恢复右侧关键点后回到 `Fusion`。

## 一致性要求

- 不再把 UVC 描述为推荐路径；UVC 只保留为可选方案。
- 启动命令必须使用仓库绝对路径或从仓库根目录开始，默认 Python 与现有脚本一致。
- 文档中的动作名称、反馈文案、按键和输出路径必须与当前代码一致。
- 不声称具备临床准确性，也不把演示结果解释为医疗结论。
